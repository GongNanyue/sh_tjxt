# 天机学堂 - 数据库设计文档

## 一、数据库总览

天机学堂采用微服务架构，每个微服务拥有独立的数据库，遵循"数据库独立"原则。系统共包含以下数据库：

| 序号 | 数据库名 | 对应微服务 | 说明 |
|------|----------|-----------|------|
| 1 | tj_user | tj-user（用户服务） | 用户账号与详情管理 |
| 2 | tj_auth | tj-auth（认证授权服务） | RBAC权限模型、登录记录 |
| 3 | tj_course | tj-course（课程服务） | 课程、分类、目录、题目管理 |
| 4 | tj_media | tj-media（媒资服务） | 文件与视频媒资管理 |
| 5 | tj_exam | tj-exam（考试服务） | 题目与考试管理 |
| 6 | tj_learning | tj-learning（学习服务） | 课表、学习记录、积分、排行榜 |
| 7 | tj_trade | tj-trade（交易服务） | 购物车、订单、退款 |
| 8 | tj_pay | tj-pay（支付服务） | 支付单、退款单 |
| 9 | tj_promotion | tj-promotion（促销服务） | 优惠券、兑换码、用户券 |
| 10 | tj-remark | tj-remark（评价服务） | 点赞记录 |
| 11 | tj_message | tj-message（消息服务） | 通知模板、短信、用户收件箱 |
| 12 | tj_search | tj-search（搜索服务） | 用户兴趣标签 |

此外还有基础设施数据库：`nacos`（注册/配置中心）、`seata`（分布式事务）、`xxl_job`（任务调度）、`gogs`（代码仓库）、`sys`（系统公共）。

## 四、表关联关系

### 4.1 用户与权限（RBAC模型）

```
user(id) ──1:1── user_detail(id)
user(id) ──1:N── account_role(account_id) ──N:1── role(id)
role(id) ──1:N── role_menu(role_id) ──N:1── menu(id)
role(id) ──1:N── role_privilege(role_id) ──N:1── privilege(id)
privilege(menu_id) ──N:1── menu(id)
```

### 4.2 课程体系

```
category(三级自关联: parent_id -> id)
course(first/second/third_cate_id) ──N:1── category(id)
course(id) ──1:N── course_catalogue(course_id)  [章->节层级: parent_catalogue_id]
course_catalogue(media_id) ──N:1── media(id)
course(id) ──1:N── course_teacher(course_id) ──N:1── user(id)  [教师]
course(id) ──1:1── course_content(id)
course(id) ──1:N── course_subject(course_id) ──N:1── subject(id)
```

### 4.3 交易链路

```
user(id) ──1:N── cart(user_id) ──N:1── course(id)
user(id) ──1:N── order(user_id) ──1:N── order_detail(order_id)
order_detail(course_id) ──N:1── course(id)
order(pay_order_no) ──1:1── pay_order(pay_order_no)
order(id) ──1:N── refund_apply(order_id) ──1:1── refund_order(biz_refund_order_no)
pay_order(pay_channel_code) ──N:1── pay_channel(channel_code)
```

### 4.4 学习体系

```
user(id) ──1:N── learning_lesson(user_id) ──N:1── course(id)
learning_lesson(id) ──1:N── learning_record(lesson_id)
learning_record(section_id) ──N:1── course_catalogue(id)
user(id) ──1:N── interaction_question(user_id) ──N:1── course(id)
interaction_question(id) ──1:N── interaction_reply(question_id)
user(id) ──1:N── points_record(user_id)
user(id) ──1:N── points_board(user_id)
```

### 4.5 促销体系

```
coupon(id) ──1:N── coupon_scope(coupon_id)
coupon(id) ──1:N── exchange_code(exchange_target_id)
coupon(id) ──1:N── user_coupon(coupon_id)
user(id) ──1:N── user_coupon(user_id)
order(coupon_ids) ──N:N── coupon(id)  [JSON存储]
```

### 4.6 考试体系

```
question(id) ──1:1── question_detail(id)
question(id) ──1:N── question_biz(question_id)
question_biz(biz_id) 关联小节id（course_catalogue.id）
```

## 六、设计特点总结

1. **微服务数据隔离**：每个服务独立数据库，避免数据耦合，通过Feign接口跨服务查询。

2. **草稿-正式双表设计**：课程模块采用草稿表（draft）与正式表分离的设计，支持课程编辑预览后再发布。

3. **Seata分布式事务**：在交易（tj_trade）、课程（tj_course）、考试（tj_exam）等涉及跨服务数据一致性的数据库中，均内置了`undo_log`表，用于Seata AT模式的事务回滚。

4. **金额单位统一为分**：所有金额字段（price, amount等）统一使用int类型、以分为单位，避免浮点精度问题。

5. **逻辑删除**：大部分核心表采用`deleted`字段实现逻辑删除，保留数据可追溯性。

6. **三级分类体系**：课程分类表通过parent_id自关联实现三级分类层次结构。

7. **JSON字段使用**：订单表中的`coupon_ids`使用JSON类型存储多个优惠券id，体现了MySQL 8.0的JSON特性。

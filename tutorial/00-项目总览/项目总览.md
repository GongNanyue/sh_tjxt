# 天机学堂（TianJi）项目总览

---

## 一、项目简介

**天机学堂**（TianJi XueTang）是一个基于 Spring Cloud 微服务架构的在线教育平台，由传智教育研究院研发。平台面向学员和教育机构，提供课程管理、在线学习、考试测评、订单交易、支付结算、促销优惠、评价互动、数据统计等完整的在线教育业务能力。

项目采用前后端分离架构，后端由 16 个微服务模块组成，通过 Spring Cloud Gateway 统一网关入口，使用 Nacos 作为注册中心和配置中心，借助 OpenFeign 实现服务间远程调用，利用 RabbitMQ 进行异步消息通信，使用 Redis/Redisson 提供缓存和分布式锁能力。

**GroupId:** `com.tianji`
**ArtifactId:** `tjxt`
**版本:** `1.0.0`
**JDK版本:** Java 11

## 六、公共组件说明（tj-common）

### 6.1 自动配置（通过 spring.factories 注册）

tj-common 通过 Spring Boot 自动配置机制向所有微服务提供以下能力：

| 配置类 | 功能说明 |
|--------|---------|
| `MqConfig` | RabbitMQ 自动配置，注入 `RabbitMqHelper` 工具类，支持同步/异步/延迟消息发送 |
| `JsonConfig` | JSON 序列化配置（Jackson），统一日期格式等 |
| `MvcConfig` | Spring MVC 配置，注册拦截器、用户上下文等 |
| `ParamCheckerConfig` | 参数校验切面配置，支持 `@ParamChecker` 注解 |
| `MybatisConfig` | MyBatis-Plus 配置，分页插件（最大200条）、动态表名插件、自动填充拦截器 |
| `RedissonConfig` | Redisson 客户端自动配置（支持单机/集群/哨兵），注入 `LockAspect` 分布式锁切面 |
| `Knife4jConfiguration` | Swagger/Knife4j 接口文档自动配置，支持响应结果自动包装 `R<T>` |
| `XxlJobConfig` | XXL-JOB 分布式任务调度执行器自动配置 |

### 6.2 统一响应封装

**`R<T>` -- 通用响应结果类**

```java
{
    "code": 200,       // 业务状态码，200-成功，其它-失败
    "msg": "OK",       // 响应消息
    "data": {...},     // 响应数据
    "requestId": "xxx" // 请求追踪ID
}
```

- `WrapperResponseBodyAdvice`：对网关转发的请求，自动将 Controller 返回值包装为 `R<T>` 格式
- 微服务间内部调用不做包装，保持原始响应，便于 Fallback 处理

### 6.3 统一异常处理

**`CommonExceptionAdvice`** -- 全局异常处理器（`@RestControllerAdvice`）

异常体系继承关系：

```
CommonException (通用异常基类, HTTP 500)
    |-- BadRequestException      (参数错误, HTTP 400)
    |-- BizIllegalException      (业务非法异常, HTTP 500)
    |-- DbException              (数据库异常, HTTP 500)
    |-- ForbiddenException       (权限不足, HTTP 403)
    |-- UnauthorizedException    (未认证, HTTP 401)
    |-- RequestTimeoutException  (请求超时)
```

异常处理策略：
- **网关请求**（来自前端）：HTTP 状态码统一返回 200，错误信息通过 `code` 字段区分
- **微服务间调用**：HTTP 状态码保持异常原始状态码，由调用方做 Fallback 处理
- 支持 Feign 异常、参数校验异常、绑定异常等多种异常类型的统一捕获

### 6.4 分布式锁（Redisson）

通过 `@Lock` 注解实现声明式分布式锁：

```java
@Lock(name = "lock:order:#{orderId}")
public void processOrder(Long orderId) { ... }
```

支持的锁策略（`LockStrategy`）：

| 策略 | 说明 |
|------|------|
| `SKIP_FAST` | 获取不到锁立即返回 false，不重试 |
| `FAIL_FAST` | 获取不到锁立即抛出异常，不重试 |
| `SKIP_AFTER_RETRY_TIMEOUT` | 重试直到超时，超时后返回 false |
| `FAIL_AFTER_RETRY_TIMEOUT` | 重试直到超时，超时后抛出异常（默认策略） |
| `KEEP_RETRY` | 不停重试直到获取锁成功 |

支持锁类型：可重入锁、公平锁、读锁、写锁。锁 Key 支持 SPEL 表达式。

### 6.5 消息队列工具（RabbitMQ）

**`RabbitMqHelper`** -- MQ 消息发送辅助类

- `send(exchange, routingKey, message)`：同步发送
- `sendDelayMessage(exchange, routingKey, message, delay)`：延迟消息发送
- `sendAsyn(exchange, routingKey, message)`：异步发送（内置线程池，核心10，最大15）

**`MqConstants`** -- MQ 常量定义，涵盖所有交换机、队列和路由键：

| 交换机 | 用途 |
|--------|------|
| `course.topic` | 课程相关事件（新建/上架/下架/过期/删除） |
| `order.topic` | 订单相关事件（支付/退款） |
| `learning.topic` | 学习相关事件（签到/学习视频/笔记等积分事件） |
| `sms.direct` | 短信发送 |
| `pay.topic` | 支付相关事件（支付成功/退款状态变更） |
| `trade.delay.topic` | 交易延迟任务（订单超时） |
| `like.record.topic` | 点赞记录（问答/笔记点赞数变更） |
| `promotion.topic` | 优惠促销事件（领券） |

### 7.5 网关鉴权机制

网关通过 `AccountAuthFilter`（全局过滤器）实现统一鉴权：

1. 获取请求方法和路径，拼接为 `METHOD:PATH` 格式
2. 判断是否为免登录路径（`/error/**`、`/jwks`、`/accounts/login` 等）
3. 解析 JWT Token 获取用户信息
4. 登录成功则将用户ID写入请求头 `user-info` 传递给下游服务
5. 校验访问权限

各业务服务可通过配置 `tj.auth.resource.enable` 启用/禁用登录拦截，并通过 `excludeLoginPaths` / `includeLoginPaths` 精细控制需要/无需登录的接口路径。

# 天机学堂 - 业务流程图

## 1. 用户注册登录流程

```mermaid
flowchart TD
    A[用户打开天机学堂] --> B{是否已登录?}
    B -- 是 --> C[进入首页]
    B -- 否 --> D[跳转登录页]
    D --> E{选择登录方式}
    E --> F[手机号+密码登录]
    E --> G[手机号+验证码登录]
    G --> G1[输入手机号]
    G1 --> G2[请求发送验证码]
    G2 --> G3[消息服务调用第三方短信平台]
    G3 --> G4[用户输入验证码]
    G4 --> G5{验证码校验}
    G5 -- 失败 --> G4
    G5 -- 成功 --> H{用户是否存在?}
    F --> F1[输入手机号和密码]
    F1 --> F2{密码BCrypt校验}
    F2 -- 失败 --> F1
    F2 -- 成功 --> H
    H -- 否 --> I[自动注册新用户]
    I --> I1[创建user记录]
    I1 --> I2[创建user_detail记录]
    I2 --> I3[分配默认学员角色]
    I3 --> J[选择兴趣分类]
    J --> K[保存interests表]
    K --> L
    H -- 是 --> L[生成JWT Token]
    L --> M[记录login_record]
    M --> N[返回Token给前端]
    N --> O[前端存储Token]
    O --> C
```

## 2. 课程购买流程

```mermaid
flowchart TD
    A[用户浏览课程列表] --> B[搜索/分类筛选课程]
    B --> C[进入课程详情页]
    C --> D{是否免费课程?}
    D -- 是 --> E[直接报名]
    E --> F[创建订单 status=5已报名]
    F --> G[发送MQ消息通知学习服务]
    G --> H[创建learning_lesson记录]
    H --> I[进入学习页面]
    D -- 否 --> J{加入购物车还是立即购买?}
    J -- 加购 --> K[添加到cart表]
    K --> L[用户继续浏览]
    L --> M[进入购物车页面]
    J -- 立即购买 --> N[选择优惠券]
    M --> N
    N --> N1[促销服务计算最优券组合]
    N1 --> O[确认订单信息]
    O --> P[创建订单 order + order_detail]
    P --> P1[订单状态: 待支付]
    P1 --> Q{选择支付渠道}
    Q --> R[微信支付]
    Q --> S[支付宝支付]
    R --> T[调用支付服务创建pay_order]
    S --> T
    T --> U[生成支付二维码]
    U --> V[用户扫码支付]
    V --> W{支付结果}
    W -- 超时未支付 --> X[订单关闭 status=3]
    W -- 支付成功 --> Y[第三方回调支付服务]
    Y --> Z[更新pay_order状态为成功]
    Z --> AA[回调交易服务]
    AA --> AB[更新order状态为已支付]
    AB --> AC[核销优惠券 user_coupon]
    AC --> AD[发送MQ消息: 订单支付成功]
    AD --> AE[学习服务接收消息]
    AE --> AF[创建learning_lesson记录]
    AF --> AG[更新user_detail购课数量]
    AG --> I
```

## 3. 课程学习流程

```mermaid
flowchart TD
    A[用户进入个人中心] --> B[查看我的课表]
    B --> C[选择要学习的课程]
    C --> D[进入课程学习页面]
    D --> E{是否制定学习计划?}
    E -- 是 --> F[设置每周学习频率]
    F --> G[保存learning_lesson.week_freq]
    E -- 否 --> H[开始学习]
    G --> H
    H --> I[查看课程目录结构]
    I --> J[选择章节/小节]
    J --> K{小节类型?}
    K -- 视频 --> L[播放视频]
    K -- 考试 --> M[进入练习/考试]
    L --> N[前端定时上报播放进度]
    N --> O[后端接收进度记录]
    O --> P[更新Redis缓存]
    P --> Q[延迟任务异步持久化到DB]
    Q --> R{是否播放完毕?}
    R -- 否 --> N
    R -- 是 --> S[标记小节已完成]
    S --> T[更新learning_record.finished=true]
    T --> U[发送MQ消息: 完成小节学习]
    U --> V[积分服务记录学习积分]
    M --> W[答题提交]
    W --> X[判断答案正确性]
    X --> Y[记录答题结果]
    Y --> U
    V --> Z{所有小节是否完成?}
    Z -- 否 --> J
    Z -- 是 --> AA[课程学习完成]
    AA --> AB[更新learning_lesson状态为已完成]
```

## 4. 视频播放进度记录流程

```mermaid
flowchart TD
    A[前端视频播放器] --> B[每15秒定时触发]
    B --> C[收集播放进度数据]
    C --> D[lessonId + sectionId + moment]
    D --> E[POST /learning-records 上报进度]
    E --> F[后端LearningRecordController接收]
    F --> G{是否首次学习该小节?}
    G -- 是 --> H[创建learning_record记录]
    G -- 否 --> I[更新已有记录]
    H --> J[写入Redis缓存]
    I --> J
    J --> K["Key: learning:record:{lessonId}"]
    K --> L["HashField: sectionId"]
    L --> M["Value: {id, moment, finished}"]
    M --> N[设置缓存过期时间1分钟]
    N --> O[提交延迟任务到DelayQueue]
    O --> P[延迟20秒后执行]
    P --> Q[从Redis读取最新缓存数据]
    Q --> R{缓存中moment与任务moment一致?}
    R -- 不一致 --> S[放弃本次持久化]
    S --> T[说明用户仍在提交新进度]
    R -- 一致 --> U[持久化到MySQL]
    U --> V[更新learning_record.moment]
    V --> W[更新learning_lesson最近学习时间]
    W --> X[更新learning_lesson最近学习小节id]

    style J fill:#f9f,stroke:#333
    style K fill:#f9f,stroke:#333
    style L fill:#f9f,stroke:#333
    style M fill:#f9f,stroke:#333
```

## 5. 点赞业务流程

```mermaid
flowchart TD
    A[用户点击点赞/取消点赞] --> B[前端发送请求到评价服务]
    B --> C{点赞还是取消?}
    C -- 点赞 --> D["SADD likes:biz:{bizId} userId"]
    C -- 取消 --> E["SREM likes:biz:{bizId} userId"]
    D --> F{操作是否成功?}
    E --> F
    F -- 失败 --> G[返回重复操作提示]
    F -- 成功 --> H["SCARD 统计点赞总数"]
    H --> I["ZADD likes:times:{bizType} bizId likedTimes"]
    I --> J[返回成功]

    K[定时任务每20秒执行] --> L["ZPOPMIN 从Redis读取点赞数据"]
    L --> M{是否有数据?}
    M -- 否 --> K
    M -- 是 --> N[转换为LikedTimesDTO列表]
    N --> O[发送MQ消息到交换机]
    O --> P[学习服务监听消息]
    P --> Q{业务类型?}
    Q -- QA --> R[更新互动问答的点赞数]
    Q -- NOTE --> S[更新笔记的点赞数]
    R --> T[批量更新数据库liked_times字段]
    S --> T

    style D fill:#f9f,stroke:#333
    style E fill:#f9f,stroke:#333
    style H fill:#f9f,stroke:#333
    style I fill:#f9f,stroke:#333
    style L fill:#f9f,stroke:#333
```

## 7. 优惠券领取流程

```mermaid
flowchart TD
    A[用户浏览优惠券列表] --> B[点击领取优惠券]
    B --> C[前端发送领券请求]
    C --> D[UserCouponController接收]
    D --> E[准备LUA脚本参数]
    E --> F["Key1: coupon:cache:{couponId}"]
    F --> G["Key2: user:coupon:{couponId}"]
    G --> H[执行receive_coupon.lua脚本]

    H --> I{LUA原子校验}
    I --> I1[校验优惠券是否存在]
    I1 --> I2[校验发放时间是否有效]
    I2 --> I3[校验库存是否充足]
    I3 --> I4[校验用户领取数量是否超限]
    I4 --> I5[Redis预扣库存]
    I5 --> I6[记录用户领取次数+1]

    I --> J{校验结果}
    J -- 优惠券不存在 --> K1[抛出异常: 优惠券不存在]
    J -- 未在发放时间 --> K2[抛出异常: 活动未开始或已结束]
    J -- 库存不足 --> K3[抛出异常: 库存不足]
    J -- 超出限领 --> K4[抛出异常: 超出领取数量]
    J -- 校验通过 --> L[发送MQ消息]

    L --> M["Exchange: promotion.exchange"]
    M --> N["RoutingKey: coupon.receive"]
    N --> O[MQ消费者异步处理]
    O --> P[checkAndCreateUserCoupon]
    P --> Q[查询优惠券详情]
    Q --> R[更新coupon已发放数量+1]
    R --> S[创建user_coupon记录]
    S --> T[设置有效期]
    T --> U[持久化到数据库]
    U --> V[领券完成]

    style H fill:#ff9,stroke:#333
    style I fill:#ff9,stroke:#333
    style I1 fill:#ff9,stroke:#333
    style I2 fill:#ff9,stroke:#333
    style I3 fill:#ff9,stroke:#333
    style I4 fill:#ff9,stroke:#333
    style I5 fill:#ff9,stroke:#333
    style I6 fill:#ff9,stroke:#333
```

## 8. 优惠券兑换码兑换流程

```mermaid
flowchart TD
    A[用户获得兑换码] --> B[在兑换页面输入兑换码]
    B --> C[前端提交兑换请求]
    C --> D[UserCouponController.exchangeCoupon]
    D --> E[CodeUtil.parseCode 解析兑换码]
    E --> F[提取序列号serialNum]
    F --> G[执行exchange_coupon.lua脚本]

    G --> H{LUA原子校验}
    H --> H1["HGET coupon:code:map 查询兑换码状态"]
    H1 --> H2[校验兑换码是否已被使用]
    H2 --> H3["ZRANGEBYSCORE 查找对应优惠券id"]
    H3 --> H4[校验优惠券发放时间]
    H4 --> H5[校验用户领取数量限制]
    H5 --> H6[标记兑换码已使用]
    H6 --> H7[预扣库存 + 记录领取次数]

    H --> I{校验结果}
    I -- 兑换码无效 --> J1[抛出异常: 兑换码不存在]
    I -- 已被使用 --> J2[抛出异常: 兑换码已被使用]
    I -- 优惠券过期 --> J3[抛出异常: 活动已结束]
    I -- 超出限领 --> J4[抛出异常: 超出领取数量]
    I -- 校验通过 --> K[返回优惠券couponId]

    K --> L[构造UserCouponDTO]
    L --> L1[设置userId, couponId, serialNum]
    L1 --> M[发送MQ消息到促销交换机]
    M --> N[MQ消费者异步处理]
    N --> O[checkAndCreateUserCoupon]
    O --> P[更新coupon已发放数量+1]
    P --> Q[创建user_coupon记录]
    Q --> R[更新exchange_code状态为已使用]
    R --> S[记录兑换用户id]
    S --> T[兑换完成]

    style G fill:#ff9,stroke:#333
    style H fill:#ff9,stroke:#333
```
